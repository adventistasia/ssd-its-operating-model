# Artifact Specifications Index

## 1. Purpose

This folder contains the normative drafting and review specifications for each canonical Work Delivery Framework artifact.

Use these files when you need to understand:

1. when an artifact is required
2. what it must contain
3. how completeness is judged
4. what common failure conditions to avoid

## 2. How to Use This Folder

1. Confirm the artifact is always required or triggered in `../artifact_catalog.md`.
2. Open the matching specification file in this folder.
3. Draft the artifact using the matching file in `../templates/`.
4. Check the artifact against `../completeness_and_review_model.md` before gate review.
