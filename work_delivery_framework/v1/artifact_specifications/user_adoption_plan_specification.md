# User Adoption Plan Specification

## 1. Purpose

The `User Adoption Plan` defines how affected users will be informed, enabled, and supported through rollout and adoption of the change.

## 2. When Required

Required when the initiative includes a customer-facing change.

## 3. Required By

Must start between Gate 4 and Gate 6 and must be complete by Gate 6.

## 4. Required Contents

1. impacted user groups and change scope
2. communications plan
3. training and enablement approach where applicable
4. rollout plan and adoption checkpoints
5. support and feedback loop plan

## 5. Completeness Rules

The plan is complete when:

1. impacted users and rollout expectations are explicit
2. support loops and feedback paths are explicit enough to avoid unmanaged adoption risk
3. the communications and enablement approach is proportionate to the change

## 6. Common Failure Conditions

1. the plan names user impact but provides no credible adoption path
2. rollout assumptions depend on undocumented communication or training actions
