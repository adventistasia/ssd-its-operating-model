# Path And Artifact Rationale

Status: working draft
Purpose: make the chosen framework path, created artifacts, and deliberately omitted artifacts inspectable.

## 1. Chosen path through the framework

Path chosen by the Delivery Owner:

`Stage 1 Work Assessment -> Stage 2 Work Definition using an Initiative Definition Document -> Stage 3 Work Authorization -> Stage 4 Work Definition Details -> Stage 5 Delivery Mobilization -> stop at execution readiness`

This path was chosen from the framework, not imposed before reading it.

## 2. Why this was escalated rather than kept as a Work Brief

Signals supporting escalation:

- formal itinerary approvals exist
- leave data is controlled by HR
- revision handling matters
- one trustworthy view is required without duplicate encoding
- there are multiple authoritative owners
- split acceptance and support ownership are visible

Delivery Owner conclusion:

- small enough to remain a governed office solution
- not small enough to fit safely into a single Work Brief without losing control clarity

## 4. Why certain common artifacts were not created

### 4.1. Work Brief

Not created because:

- the work moved beyond one controlled record once HR leave data and formal itinerary authority were both in scope
- one Work Brief would have been too thin for the split ownership and split acceptance story

### 4.2. Detailed technical design

Not created because:

- the exercise stops at execution readiness
- detailed build and deployment design would overstate the level of simulation

### 4.3. Deployment / live-service evidence

Not created because:

- no build, deployment, or operational go-live is being simulated
- the scenario stops intentionally before execution

## 5. Main judgment on proportionality

The created set is intentionally richer than scenario 01:

- formal enough to test the framework
- detailed enough to show why the request escalated
- still bounded enough to stay believable for an office tracker request

That unevenness is part of the test, not a defect in the scenario packaging.
